                                                        SVG
                                                Scalable Vector Graphics

Стилізація SVG

    <svg width="100" height="100">
    <rect width="90" height="90" x="5" y="5" style="fill: blue; stroke: black; stroke-width: 5; opacity: 0.9" />
    </svg>

    У цьому коді ми визначаємо SVG область розміром 100x100 пікселів і створюємо всередині неї прямокутник з наступними стилями:
        fill: blue - заливаємо прямокутник синім кольором.
        stroke: black - задаємо колір обводки чорним.
        stroke-width: 5 - встановлюємо товщину обводки в 5 пікселів.
        opacity: 0.9 - задаємо прозорість зображення на рівні 90%.


Вставка SVG

    1. Використання тегу <img>:
        Простий і швидкий спосіб, але без можливості маніпуляції її частинами через CSS або JavaScript.
        Підходить для статичних зображень, де не потрібна взаємодія чи анімація (наприклад, лого).

        Приклад:
            <img src="path-to-image/image.svg" alt="Image Description">


    2. Інлайново через <svg>.
        Вставка SVG коду безпосередньо в HTML дає найбільшу гнучкість, дозволяючи стилізувати та маніпулювати кожною частиною SVG за допомогою CSS і JavaScript.
        Ідеально підходить для інтерактивних або анімованих векторних графік.

        Приклад:
            <svg width="100" height="100">
            <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
            </svg>

    3. Повторно через <use>

    Елемент <symbol>
        Використовується для оголошення графічних об'єктів, які можна повторно використовувати в SVG документі за допомогою елемента <use>.
        Елементи <symbol> не відображаються в документі до тих пір, поки не будуть викликані через <use>, що робить їх ідеальними для визначення "шаблонів" графіки, які потрібно використовувати кілька разів.

    Елемент <use>
        Використовується для створення копій SVG елементів, які були визначені раніше за допомогою <symbol> або безпосередньо в <svg>.
        Це дозволяє нам вставляти копії оригінальних елементів без необхідності копіювати їх код, спрощуючи управління та оновлення зображень.

        Приклад:
            <svg style="height: 0">
                <symbol id="icon-heart" viewBox="0 0 100 100">
                    <!-- SVG контент для іконки "серце" -->
                    <path d="M10,30 ... Z" fill="red"/>
                </symbol>
            </svg>

            <!-- Використання іконки "серце" у документі -->
            <svg width="100" height="100">
                <use href="#icon-heart"></use>
            </svg>

            <svg width="50" height="50">
                <use href="#icon-heart"></use>
            </svg>


SVG sprite
    Файл, в якому зберігаються іконки у вигляді <symbol>, які потім підключаються через <use>.
    Це найчистіший і найправильніший спосіб працювати з іконками.

    Приклад:
    - sprite.svg:
        <svg xmlns="http://www.w3.org/2000/svg" style="display: none;">
            <symbol id="icon-arrow" viewBox="0 0 24 24">
                <path d="M5 12h14M13 6l6 6-6 6" fill="currentColor"/>
            </symbol>

            <symbol id="icon-user" viewBox="0 0 24 24">
                <path d="M12 12a5 5 0 1 0-0.001-10.001A5 5 0 0 0 12 12z" fill="currentColor"/>
            </symbol>
        </svg>

    - використання іконки:
        <svg class="icon">
            <use href="sprite.svg#icon-arrow"></use>
        </svg>

    - стилізація через CSS:
        .icon {
            width: 24px;
            height: 24px;
            fill: black;
            stroke: black;
        }
    
    Важливо:
    SVG іконки краще робити з fill="currentColor" всередині — тоді вони автоматично підлаштовуються під color.

        .icon {
        color: red;
        }

    Часті проблеми
        ❌ Не працює стилізація?
            Перевір:
            Чи немає в <path> жорстко прописаного fill="#000"
            Якщо є — видали або заміни на currentColor

        ❌ Чи можна задати клас всередині sprite?
            Ні.
            sprite.svg — це окремий документ.
            Ти стилізуєш зовнішній <svg>, а не <symbol> напряму.


    Чи треба ::before / ::after?
        Якщо іконка декоративна — можна вставити через ::before / ::after:
            button::before {
            content: "";
            background: url("icon.svg");
            }

        Але для керованих іконок краще <svg><use>.
