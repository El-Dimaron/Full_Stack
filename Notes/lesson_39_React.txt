Компоненти

  Компоненти - це основні будівельні блоки додатків на React. Компоненти створюються як незалежні, ізольовані частини інтерфейсу з власною бізнес-логікою та станом.

  Компоненти поділяються на два основні типи: компоненти зі станом (container або stateful components) та презентаційні компоненти (presentational або stateless components).

    Компоненти зі станом управляють даними за допомогою хуків і використовуються, коли потрібно зберігати або змінювати інформацію на основі взаємодії користувача або зовнішніх даних.

    Презентаційні компоненти відповідають лише за відображення даних, які отримують через props. Вони не мають власного стану, що робить їх простими та легкими для тестування.


  React-компонент складається з трьох основних частин:
    JavaScript: логіка компонента (стан, обробка подій)
    JSX: розмітка компонента (структура)
    CSS: стилізація компонента (зовнішній вигляд)

    // Приклад
    import { useState } from 'react';
    import './Button.css'; // CSS стилі

    const Button = ({ text }) => {
      // JavaScript: логіка
      const [isActive, setIsActive] = useState(false);
      const handleClick = () => setIsActive(!isActive);

      // JSX: структура з CSS класами
      return (
        <button
          className={`button ${isActive ? 'active' : ''}`}
          onClick={handleClick}
        >
          {text}
        </button>
      );
    };


  Односпрямований потік (Unidirectional flow)

    Це ключова концепція у React, яка визначає, як дані передаються між компонентами. Вона означає, що дані у React передаються лише від батьківських компонентів до дочірніх через властивості (props).

    Дочірній компонент може читати ці властивості, але не може їх змінювати - це гарантує, що потік даних завжди йде в одному напрямку. Такий підхід допомагає керувати станом програми та спрощує розуміння того, як дані змінюються в часі.


Controlled and Uncontrolled Components

  Uncontrolled Components
    Неконтрольовані компоненти в React — це компоненти форми, у яких поточне значення поля зберігається та змінюється самим DOM-елементом, а не керується React через value і onChange. Для доступу до цього значення зазвичай використовують ref.

    Як працюють неконтрольовані компоненти
      1. Зберігання даних: Дані форм зберігаються в DOM, а не в стані компонента.
      2. Використання ref: Для доступу до значень елементів форми використовується useRef, який надає посилання на DOM-елемент.
      3. Події форми: Обробка подій форми відбувається через обробники подій, а значення зчитуються через ref або об'єкт event.
      4. Роль об'єкта event: Об'єкт event дозволяє отримати доступ до значень елементів форми через event.target.elements і запобігти стандартній поведінці форми за допомогою event.preventDefault().


    Хук useRef
      Хук useRef у React використовується для отримання прямого доступу до DOM-елементу або для зберігання будь-якої мутабельної інформації, яка не викликає перерендер компонента при її зміні.

      Приклад коду для неконтрольованого компонента з використанням useRef:
        import { useRef } from 'react';

        const UncontrolledComponent = () => {
          const inputRef = useRef(null); // Створення ref для доступу до input

          const handleSubmit = (event) => {
            event.preventDefault(); // Запобігання перезавантаженню сторінки
            alert('Введене ім\'я: ' + inputRef.current.value); // Отримання значення через ref
          };

          return (
            <form onSubmit={handleSubmit}>
              <label>
                Ім'я:
                <input type="text" ref={inputRef} /> {/* Прив'язка ref до input */}
              </label>
              <button type="submit">Надіслати</button>
            </form>
          );
        };

        export default UncontrolledComponent;

  Controlled Components
    Контрольовані компоненти — це елементи форми (<input>, <textarea>, <select>), у яких React керує поточним значенням через state. Значення елемента завжди відповідає значенню зі стану компонента, що реалізує принцип “єдиного джерела істини”. Кожна зміна в полі викликає onChange, який оновлює state через setState/set-функцію, а оновлений state, у свою чергу, передається назад в елемент через value.

    Для controlled input має бути такий звʼязок:
      const [name, setName] = useState("");

      <input
        value={name}
        onChange={(event) => setName(event.target.value)}
      />

    Тут:
      value={name}
    Означає:
      input показує те значення, яке лежить у React state.

    А:
      onChange={(event) => setName(event.target.value)}
    Означає:
      коли користувач щось вводить, ми оновлюємо React state.

    І виходить коло:
      Користувач вводить текст
      ↓
      onChange отримує нове значення
      ↓
      setName оновлює state
      ↓
      React ререндерить компонент
      ↓
      input бере нове значення з value={name}


    Переваги
      Миттєвий доступ до значення поля
      Можливість валідації при введенні
      Централізоване керування даними форми
      Легке скидання або зміна значень програмно
      Легка інтеграція з бібліотеками для управління станом (наприклад, Redux, Formik)

    Недоліки
      Потребує більше коду порівняно з неконтрольованими компонентами
      Необхідність створення обробника для кожного поля
      Додаткові ререндери при кожній зміні значення, що може вплинути на продуктивність у великих формах


  Controlled vs Uncontrolled Components
    Вибір між контрольованими та неконтрольованими компонентами у React залежить від конкретного сценарію використання:

    Використання контрольованих компонентів:
      - Дані форми потребують негайної валідації: Контрольовані компоненти підходять для сценаріїв з валідацією введення користувача в реальному часі (перевірка формату електронної пошти).
      - Динамічне змінення введених даних: Для зміни введення на основі інших дій у формі (автоматичне форматування номера телефону) контрольовані компоненти забезпечують необхідний контроль.
      - Поділ стану між компонентами: Для поділу стану форми між декількома компонентами або використання даних форми в інших частинах додатку доцільно використовувати контрольовані компоненти.

    Використання неконтрольованих компонентів:
      - Інтеграція зі сторонніми DOM-бібліотеками: Для інтеграції React з не-React бібліотеками, що керують власним станом, неконтрольовані компоненти спрощують процес.
      - Прості форми: Для форм без складної взаємодії або валідації на клієнтській стороні неконтрольовані компоненти зменшують обсяг коду та підвищують продуктивність.
      - Обробка великих форм: У випадках з великими формами неконтрольовані компоненти можуть працювати швидше через меншу кількість рендерингів та оновлень стану.

    Рекомендації:
      - Надавати перевагу контрольованим компонентам для форм, що потребують взаємодії з іншими частинами додатку або валідації даних на стороні клієнта.
      - Застосовувати неконтрольовані компоненти для простих форм або роботи з великими даними, де продуктивність критично важлива.
      - Обирати підхід на основі вимог додатку, враховуючи читабельність, підтримку та продуктивність коду.


State

  Стан використовується для зберігання даних, що можуть змінюватися. Компонент може змінювати свій стан за допомогою:
    - функції setState (для класових компонентів)
    - хука useState (для функціональних компонентів)

  Коли стан змінюється, компонент автоматично перерендерується.


useState
  Хук useState є інструментом React для роботи зі станом у функціональних компонентах. Він дозволяє створювати та змінювати дані, які мають викликати оновлення інтерфейсу.

  Примітка: У React звичайні змінні не викликають оновлення інтерфейсу при їх зміні. Для відстеження змін та автоматичного оновлення компонентів використовується спеціальний механізм стану - useState.

  // Приклад використання useState для створення лічильника
    import { useState } from 'react';

    const Counter = () => {
      // useState приймає початкове значення (0) і повертає масив з двох елементів:
      // count - поточне значення стану
      // setCount - функція для оновлення стану
      const [count, setCount] = useState(0);

      // Функція-обробник збільшує значення на 1 при кожному кліку
      const handleClick = () => {
        setCount(count + 1); // Викликає перерендер компонента
      };

      return (
        <div>
          <p>Ви натиснули {count} разів</p>
          <button onClick={handleClick}>
            Натисніть тут, щоб збільшити лічильник
          </button>
        </div>
      );
    };

    export default Counter;

    У цьому прикладі створено компонент Counter, який використовує useState для відстеження кількості натискань на кнопку. Хук useState повертає масив з двох елементів: поточне значення стану (count) та функцію для його оновлення (setCount). При кожному натисканні на кнопку викликається функція handleClick, яка через setCount збільшує значення count на одиницю.


prevState
  При оновленні стану в React важливо враховувати його попереднє значення. Використання prevState гарантує, що ми працюємо з актуальним станом, особливо коли відбувається декілька оновлень поспіль.

    import { useState } from 'react';

    const Counter = () => {
      const [count, setCount] = useState(0);

      // Неправильно: може призвести до втрати оновлень
      const incrementWrong = () => {
        setCount(count + 1);
        setCount(count + 1); // Все ще використовує початкове значення count
      };

      // Правильно: використовує prevState для надійного оновлення
      const incrementRight = () => {
        setCount(prevCount => prevCount + 1);
        setCount(prevCount => prevCount + 1);
      };

      return (
        <div>
          <h1>Лічильник: {count}</h1>
          <button onClick={incrementWrong}>
            Додати 2 (неправильно)
          </button>
          <button onClick={incrementRight}>
            Додати 2 (правильно)
          </button>
        </div>
      );
    };

  export default Counter;
  Помилка
  У цьому прикладі incrementWrong не гарантує збільшення на 2, оскільки обидва виклики setCount використовують те саме початкове значення count. Натомість, incrementRight використовує prevState, що забезпечує коректне послідовне оновлення стану. Це важливо для надійної роботи компонента при багаторазових або асинхронних оновленнях стану.

useEffect

  Базова інформація
    useEffect – це хук у React, який дозволяє виконувати побічні ефекти в функціональних компонентах. Використовується для виконання дій, які потребують взаємодії з DOM, запитів до API, підписок або відписок та інших операцій, що не повинні виконуватися під час кожного рендеру. useEffect викликається після кожного завершеного рендерингу, але може бути налаштований для запуску лише при зміні певних значень.

    Приклад використання useEffect для завантаження даних з API:
      import { useState, useEffect } from 'react';

      const UserData = () => {
          // Створення стану для збереження даних користувача
          const [user, setUser] = useState(null);

          // Виконання запиту до API після монтування компонента
          useEffect(() => {
              // Функція для отримання даних
              const fetchData = async () => {
                  try {
                      const response = await fetch('https://jsonplaceholder.typicode.com/users/1');
                      const data = await response.json();
                      setUser(data); // Збереження отриманих даних
                  } catch (error) {
                      console.error('Помилка отримання даних:', error);
                  }
              };

              fetchData();
          }, []); // Пустий масив залежностей - ефект виконається один раз

          return (
              <div>
                  {user ? (
                      // Відображення даних користувача
                      <div>
                          <p>Ім'я: {user.name}</p>
                          <p>Email: {user.email}</p>
                      </div>
                  ) : (
                      // Відображення стану завантаження
                      <p>Завантаження даних користувача...</p>
                  )}
              </div>
          );
      };

      export default UserData;

  Масив залежностей useEffect
    Масив, який передається в useEffect, відомий як масив залежностей. Він контролює момент виконання ефекту на основі зміни вказаних значень. За порожнього масиву залежностей ефект виконується лише один раз після першого рендеру компонента. При наявності змінних у масиві, ефект спрацьовує при кожній їх зміні.

    Приклад компонента, що відображає кількість лайків статті та дозволяє їх змінювати:
      import { useState, useEffect } from 'react';

      const LikeCounter = () => {
          // Стан для збереження кількості лайків
          const [likes, setLikes] = useState(0);
          // Стан для збереження повідомлення
          const [message, setMessage] = useState('');

          useEffect(() => {
              // Ефект виконується при зміні кількості лайків
              console.log(`Кількість лайків змінилася на ${likes}`);
              setMessage(`Віддано ${likes} лайків`);
          }, [likes]); // Залежність від змінної likes

          // Обробник кліку для збільшення лайків
          const handleLikeClick = () => setLikes(likes + 1);

          return (
              <div>
                  <button onClick={handleLikeClick}>Лайк</button>
                  <p>{message}</p>
              </div>
          );
      };

      export default LikeCounter;

  Асинхронні операції
    Під час використання useEffect з асинхронними операціями не можна робити саму callback-функцію ефекту асинхронною напряму. Тому всередині useEffect створюють окрему async-функцію, наприклад fetchData, і викликають її одразу після оголошення.

    Для обробки помилок асинхронний код зазвичай обгортають у try...catch. Якщо помилку потрібно показати в інтерфейсі, додатково створюють state для її зберігання, наприклад error/setError, і відображають повідомлення в JSX.

      useEffect(() => {
      const fetchData = async () => {
        try {
          const response = await fetch(url);

          if (!response.ok) {
            throw new Error("Something went wrong");
          }

          const data = await response.json();
          setData(data);
        } catch (error) {
          setError(error.message);
        }
      };

      fetchData();
    }, []);


    Головне уточнення:
      useEffect(async () => {
        // так не треба
      }, []);

      Не рекомендується, бо useEffect очікує, що callback поверне або undefined, або cleanup-функцію. А async-функція завжди повертає Promise.

Повторний рендеринг React-компонента
  У React, коли властивості (props) або стан (state) компонента змінюються, компонент автоматично оновлюється на сторінці. React сам відслідковує зміни в даних і оновлює відображення за потреби.


Умовний рендеринг (conditional rendering)
  Умовний рендеринг дозволяє відображати різні компоненти або елементи залежно від певних умов. React пропонує декілька способів реалізації умовного рендерингу.

  Три основні підходи до умовного рендерингу:
    - Тернарний оператор (? :) - для простого вибору між двома варіантами
    - Логічний оператор && - для відображення/приховування елемента
    - Функція з if/else - для складної логіки з багатьма умовами

  Приклад:
    import { useState } from 'react';

    const ConditionalDemo = () => {
      const [isLoggedIn, setIsLoggedIn] = useState(false);
      const [isAdmin, setIsAdmin] = useState(false);
      const [notifications, setNotifications] = useState(['Повідомлення 1']);

      // 1. Тернарний оператор (для простих умов)
      const greeting = isLoggedIn
        ? <h1>Вітаємо!</h1>
        : <h1>Увійдіть в систему</h1>;

      // 2. Логічний оператор && (для відображення/приховування)
      const notificationsList = notifications.length > 0 && (
        <div>У вас {notifications.length} повідомлень</div>
      );

      // 3. If/else в функції (для складної логіки)
      const getAdminPanel = () => {
        if (!isLoggedIn) return null;
        if (!isAdmin) return <p>Немає прав доступу</p>;
        return <div>Панель адміністратора</div>;
      };

      return (
        <div>
          {/* Тернарний оператор */}
          {greeting}

          {/* Логічний оператор && */}
          {notificationsList}

          {/* Виклик функції */}
          {getAdminPanel()}
        </div>
      );
    };


Кореневий елемент у JSX
  У JSX коді може бути лише один кореневий елемент. Це обмеження зумовлене синтаксисом JSX та його перетворенням у виклики React.createElement. При написанні JSX коду кожен компонент повинен мати один обгортковий елемент, який слугує контейнером для всіх інших елементів всередині компонента.

  React Fragment (<>...</> або <Fragment>...</Fragment>) використовується для групування декількох елементів JSX без додавання додаткових вузлів до DOM.

    Існує два способи використання Fragment:

    // Використання <>...</>
    <>
      <ChildA />
      <ChildB />
    </>

    // Використання <Fragment></Fragment>
    <Fragment>
      <ChildA />
      <ChildB />
    </Fragment>

    Повний синтаксис <Fragment> необхідний, оскільки при рендерингу списків потрібно вказувати атрибут key.


Передача даних між батьківським і дочірніми елементами
  Підняття стану (Lifting State Up)
    Підняття стану вгору у React – це процес переміщення стану з дочірніх компонентів до їх спільного батьківського компонента. Цей підхід використовується, коли кілька дочірніх компонентів потребують доступу до одних і тих же даних або коли потрібно синхронізувати стан між ними. Замість того, щоб тримати локальний стан в кожному дочірньому компоненті, стан розташовується у батьківському компоненті та передається дочірнім через пропси.

    import { useState } from 'react';

    // Батьківський компонент
    const ParentComponent = () => {
      // Стан та функції його зміни знаходяться в батьківському компоненті
      const [count, setCount] = useState(0);

      // Ці функції передаються дочірнім компонентам через props
      const incrementCount = () => {
        setCount(count + 1); // Оновлення стану впливає на всі дочірні компоненти
      };

      const decrementCount = () => {
        setCount(count - 1);
      };

      return (
        <div>
          <h1>Загальний лічильник: {count}</h1>
          {/* Обидва дочірні компоненти мають доступ до одного стану */}
          <ChildCounter
            count={count}
            onIncrement={incrementCount}
            label="Перший лічильник"
          />
          <ChildCounter
            count={count}
            onDecrement={decrementCount}
            label="Другий лічильник"
          />
        </div>
      );
    };

    // Дочірній компонент
    const ChildCounter = ({ count, onIncrement, onDecrement, label }) => {
      return (
        <div>
          <h2>{label}: {count}</h2>
          {onIncrement && (
            <button onClick={onIncrement}>Збільшити</button>
          )}
          {onDecrement && (
            <button onClick={onDecrement}>Зменшити</button>
          )}
        </div>
      );
    };

    export default ParentComponent;


  Стан через властивості (State via Props)
    Стан через властивості у React – це підхід, за допомогою якого передається стан з батьківського компонента до дочірнього через пропси. Дочірній компонент отримує дані через пропси і може їх відображати.

    import { useState } from 'react';

    // Батьківський компонент створює та зберігає стан
    const ParentComponent = () => {
      // Створюємо стан повідомлення
      const [message, setMessage] = useState('Привіт від батьківського компонента');

      return (
        <div>
          {/* Передаємо стан дочірньому компоненту через проп message */}
          <ChildComponent message={message} />
        </div>
      );
    };

    // Дочірній компонент отримує дані через проп message
    const ChildComponent = ({ message }) => {
      return (
        <div>
          <p>{message}</p>
        </div>
      );
    };

    export default ParentComponent;
