Об'єкт window
  Об'єкт window є глобальним об'єктом у середовищі виконання JavaScript у веб-браузерах. Він представляє вікно браузера та містить властивості та методи, які дозволяють взаємодіяти з браузером і його документом. Цей об'єкт включає в себе функціонал для керування вікном браузера, обробки подій, зберігання даних та багато іншого.

  Приклади коду:
    Відкриття нового вікна браузера з URL-адресою Google
    window.open('https://www.google.com', '_blank');

    Встановлення таймеру з використанням setTimeout:
      window.setTimeout(function() {
        alert('Привіт через 2 секунди!');
      }, 2000);

    Робота з локальним сховищем (LocalStorage):
      // Зберігаємо дані в локальному сховищі
        window.localStorage.setItem('ключ', 'значення');

      // Отримуємо дані з локального сховища
        const value = window.localStorage.getItem('ключ');
        console.log(value); // Виводить 'значення'

      Локальне сховище (localStorage) дозволяє зберігати пари ключ-значення в браузері користувача, що забезпечує можливість збереження даних між сесіями браузера.

    Перехід за адресою (URL):
      // Перенаправляє користувача на іншу сторінку
        window.location.href = 'https://www.example.com';

      Використання властивості location.href дозволяє змінити поточну адресу браузера, що призводить до перенаправлення користувача на нову веб-сторінку.


BOM
  Об'єктна модель браузера, або Browser Object Model (BOM), це колекція об'єктів, які надають функціонал для взаємодії з веб-браузером. Вона не має стандартної специфікації, але всі сучасні браузери підтримують стандартний набір об'єктів. BOM дає нам можливість працювати з частинами браузера за межами вмісту веб-сторінок, такими як адресна стрічка, історія, вікна, екрани, і більше. Ми можемо використовувати BOM для виконання таких завдань, як маніпуляція з вікнами, взаємодія з веб-сервером через AJAX запити, зберігання даних на клієнтській стороні та багато іншого.

  Screen
    Об'єкт screen у веб-браузері надає інформацію про екран, на якому відображається браузер. Цей об'єкт містить властивості, такі як ширина та висота екрану, глибина кольору та інші, що дозволяють нам отримувати різноманітні дані про екран користувача.

  Navigator
    Об'єкт navigator є частиною Browser Object Model (BOM) та містить інформацію про веб-браузер, який використовує користувач. Цей об'єкт дозволяє нам отримувати дані про клієнтське програмне забезпечення, як-от версію браузера, ім'я браузера, мовні налаштування, підтримку певних технологій і багато іншого. Ми використовуємо navigator для адаптації функціональності або вмісту веб-додатків з урахуванням можливостей та налаштувань браузера користувача.
      const browserName = navigator.appName;
      const browserVersion = navigator.appVersion;

      console.log(`Браузер: ${browserName}, Версія: ${browserVersion}`);

  History
    Об'єкт history є частиною Browser Object Model (BOM) і відповідає за історію переходів в рамках сесії поточного вікна або вкладки браузера. Ми можемо використовувати методи об'єкта history для навігації вперед і назад по історії браузера, а також для маніпулювання історією сесії. Це дає нам можливість програмно змінювати стан історії без перезавантаження сторінки.

    Приклади коду:
    1. Перехід назад у історії браузера:
      // Повертає користувача на одну сторінку назад
      history.back();

    2. Перехід вперед у історії браузера:
      // Переміщує користувача на одну сторінку вперед, якщо він раніше перейшов назад
      history.forward();

    3. Перехід на конкретну сторінку в історії браузера:
      // Переміщує користувача на дві сторінки назад у історії
      history.go(-2);

    4. Додавання стану в історію браузера:
      // Додає новий запис до історії сесії без перезавантаження сторінки
      history.pushState({ page: 1 }, "title 1", "?page=1");

    5. Заміна поточного стану в історії браузера:
      // Замінює поточний запис у історії на новий
      history.replaceState({ page: 2 }, "title 2", "?page=2");

  Frames
    Об'єкт frames у веб-браузері використовується для доступу до фреймів на сторінці. Фрейми — це частини сторінки, які можуть містити окремі документи, кожен з яких має свій власний контекст виконання. У минулому фрейми були популярні для вбудовування змісту від інших джерел за допомогою тегів <frame> та <iframe>. Об'єкт frames дозволяє отримувати доступ до цих вбудованих вікон та керувати ними.


  Location
    Об'єкт location використовується для отримання інформації про поточну URL-адресу веб-сторінки та для маніпуляції нею. Він дозволяє нам отримувати різні компоненти URL, такі як хост, шлях, рядок запиту (query string), протокол тощо, а також містить методи для перенаправлення на іншу сторінку або перезавантаження поточної сторінки.

    Приклад коду:
      // Отримання повного URL поточної сторінки
      const currentUrl = window.location.href;
      console.log(`Поточний URL: ${currentUrl}`);

      // Зміна шляху на поточній сторінці (без перезавантаження)
      window.location.pathname = '/new-path';

      // Додавання параметрів до URL
      window.location.search = '?query=новий_запит';

      // Перенаправлення на нову сторінку
      window.location.href = 'https://example.com';

      // Перезавантаження поточної сторінки
      window.location.reload();

  Fetch
    Метод fetch використовується для асинхронного здійснення HTTP-запитів до сервера з метою отримання або відправлення даних. Він є частиною Fetch API, яке надає зручний та потужний інтерфейс для роботи з мережевими запитами. fetch повертає обіцянку (Promise), що дозволяє використовувати асинхронний код для обробки відповіді або помилок без блокування виконання програми.

  Cookies
    Cookies (кукі) - це невеликі фрагменти даних, які веб-сайти зберігають на комп'ютері користувача через веб-браузер. Вони використовуються для збереження інформації про сесію користувача, налаштування сторінок або стан входу в систему. Cookies дозволяють веб-сайтам "пам'ятати" користувачів при повторних відвідуваннях або під час навігації між сторінками веб-сайту, тим самим покращуючи їх досвід користування.

      Приклад коду:
        // Збереження cookie
          function setCookie(name, value, days) {
            let expires = "";
            if (days) {
              const date = new Date();
              date.setTime(date.getTime() + (days * 24 * 60 * 60 * 1000));
              expires = "; expires=" + date.toUTCString();
            }
            document.cookie = name + "=" + (value || "")  + expires + "; path=/";
          }

        // Отримання cookie
          function getCookie(name) {
            const nameEQ = name + "=";
            const ca = document.cookie.split(';');
            for(let i = 0; i < ca.length; i++) {
              let c = ca[i];
              while (c.charAt(0) === ' ') c = c.substring(1, c.length);
              if (c.indexOf(nameEQ) === 0) return c.substring(nameEQ.length, c.length);
            }
            return null;
          }

        // Встановлення cookie
          setCookie('user', 'John Doe', 7);

          // Отримання значення cookie
          const user = getCookie('user');
          console.log('Користувач:', user);

  Web Storage API
    Web Storage API надає два механізми для зберігання даних в браузері: localStorage та sessionStorage. Ці механізми дозволяють веб-сайтам зберігати дані у вигляді пар ключ-значення на стороні клієнта. Відмінність між ними полягає в області видимості та тривалості зберігання: localStorage зберігає дані без обмеження терміну, доки вони вручну не будуть видалені, тоді як sessionStorage зберігає дані лише протягом сесії браузера (дані втрачаються при закритті вкладки).

    Приклад коду:
      Зберігання даних за допомогою localStorage:
        // Зберігання пари ключ-значення
        localStorage.setItem('user', 'John Doe');

        // Отримання значення за ключем
        const user = localStorage.getItem('user');
        console.log('Збережений користувач:', user);

        // Видалення збереженого значення за ключем
        localStorage.removeItem('user');

      Зберігання даних за допомогою sessionStorage:
        // Зберігання пари ключ-значення
        sessionStorage.setItem('sessionData', 'Тимчасові дані');

        // Отримання значення за ключем
        const sessionData = sessionStorage.getItem('sessionData');
        console.log('Збережені тимчасові дані:', sessionData);

        // Видалення збереженого значення за ключем
        sessionStorage.removeItem('sessionData');


Popup Boxes
  alert, confirm та prompt є методами об'єкта window у веб-браузері, які використовуються для взаємодії з користувачем через спливаючі вікна.

  - alert
    використовується для відображення повідомлення користувачу. Воно містить кнопку OK, що дозволяє користувачу закрити вікно.
  - confirm
    використовується для відображення повідомлення з двома кнопками: OK та Cancel. Воно повертає булеве значення в залежності від того, яку кнопку натиснув користувач: true для OK і false для Cancel.
  - prompt
    використовується для отримання введення від користувача. Він відображає спливаюче вікно з текстовим полем для введення даних, кнопкою OK для підтвердження та кнопкою Cancel для відмови. Метод повертає рядок, введений користувачем, або null, якщо користувач натиснув Cancel.


Timers
  Таймери в JavaScript дозволяють нам виконувати код через заданий інтервал часу або затримувати його виконання. Існують два основні методи для роботи з таймерами: setTimeout для виконання коду один раз після затримки та setInterval для повторюваного виконання коду через рівні проміжки часу.

  Приклад коду:
    Використання setTimeout:
      // Встановлення таймера, який виконує функцію через 2000 мілісекунд (2 секунди)
      setTimeout(function() {
        console.log('Привіт через 2 секунди!');
      }, 2000);

      Цей код встановлює таймер, який виведе повідомлення "Привіт через 2 секунди!" на консоль після затримки в 2 секунди.

  Використання setInterval:
    // Встановлення інтервального таймера, який виводить повідомлення кожні 1000 мілісекунд (1 секунду)
    const intervalId = setInterval(function() {
      console.log('Це повідомлення з\'являтиметься кожну секунду');
    }, 1000);

    // Зупинка інтервального таймера через 5 секунд
    setTimeout(function() {
      clearInterval(intervalId);
      console.log('Інтервал зупинено');
    }, 5000);

DOM
  General
    Об'єктна модель документа, або Document Object Model (DOM), це програмний інтерфейс, який дозволяє нам читати та змінювати вміст веб-сторінок. DOM представляє документ як структуроване дерево вузлів та об'єктів, що дає можливість програмам змінювати текст, структуру, стиль документа. Коли веб-сторінка завантажується у браузер, вона стає доступною через об'єкт document.

    У Document Object Model (DOM) існує 12 типів вузлів, кожен з яких має унікальний номерний ідентифікатор.

    Найбільш актуальні вузли:
      1. ELEMENT_NODE (1) - це вузли, які представляють HTML-теги на сторінці. Вони є основними будівельними блоками веб-сторінок і можуть містити інші елементи, текстові вузли або коментарі як своїх дітей.
      2. TEXT_NODE (3) - містять текст, який є вмістом HTML-елементів. Кожен фрагмент тексту у документі обгорнутий текстовим вузлом, що дозволяє маніпулювати текстом за допомогою DOM.
      3. COMMENT_NODE (8) - це вузли, які містять коментарі з HTML-коду. Хоча вони не відображаються як частина веб-сторінки, вони присутні в DOM-дереві і можуть бути використані для додавання пояснень або інструкцій у коді.
      4. DOCUMENT_NODE (9) - це кореневий вузол, який слугує точкою входу до DOM веб-сторінки. Він представляє весь документ і надає основні методи та властивості для доступу та маніпуляції вмістом.

      Ці типи вузлів є найбільш часто використовуваними в практиці роботи з DOM, надаючи можливість розробникам ефективно створювати, змінювати та взаємодіяти з вмістом веб-сторінок.

  DOM методи та властивості
    Ці методи та властивості є частиною основного інструментарію для роботи з DOM у JavaScript. Вони дозволяють ефективно вибирати, модифікувати, додавати та видаляти елементи на веб-сторінці:

    querySelector
      Повертає перший елемент, що відповідає CSS-селектору.
      const element = document.querySelector('.class #id');

    querySelectorAll
      Повертає статичну колекцію всіх елементів, що відповідають CSS-селектору.
      const elements = document.querySelectorAll('div.class');

    getElementById
      Повертає елемент за унікальним ID.
      const element = document.getElementById('unique-id');

    getElementsByClassName
      Повертає живу колекцію елементів з заданим класом.
      const elements = document.getElementsByClassName('some-class');

    getElementsByTagName
      Повертає живу колекцію елементів за назвою тегу.
      const elements = document.getElementsByTagName('p');

    getElementsByName
      Повертає NodeList елементів за значенням атрибута name.
      const elements = document.getElementsByName('name-attribute');

    getAttribute
      Повертає значення атрибута елемента.
      const attributeValue = element.getAttribute('href');

    setAttribute
      Встановлює або змінює значення атрибута елемента.
      element.setAttribute('href', 'https://example.com');

    className
      Повертає або встановлює значення атрибута class елемента.
      element.className = 'new-class-name';

    classList
      Надає методи для роботи з класами елемента.
      element.classList.add('new-class');
      element.classList.remove('old-class');
      element.classList.toggle('toggle-class');
      element.classList.contains('some-class'); // повертає true або false

    removeChild
      Видаляє дочірній вузол з батьківського елемента.
      const child = document.getElementById('child-id');
      element.removeChild(child);

    appendChild
      Додає вузол в кінець списку дочірніх елементів батьківського елемента.
      const newElement = document.createElement('div');
      element.appendChild(newElement);

    insertBefore
      Вставляє новий вузол перед існуючим дочірнім вузлом.
        const newElement = document.createElement('div');
        element.insertBefore(newElement, element.firstChild); // Вставити перед першим дочірнім елементом

    replaceChild
      Замінює один дочірній вузол іншим.
      const newElement = document.createElement('div');
      const oldElement = document.getElementById('old-id');
      element.replaceChild(newElement, oldElement);

    cloneNode
      Створює копію вузла, та повертає клонований вузол.
      const clone = element.cloneNode(true); // якщо true, клонується з усіма дочірніми елементами


  HTMLCollection vs NodeList
    Два типи колекцій, які використовуються для зберігання наборів DOM вузлів. Обидва є "подібними до масивів" (array-like), але не є справжніми JavaScript масивами і мають різні методи та властивості.

    HTMLCollection:
      Жива колекція, яка автоматично оновлюється при зміні DOM.
      Містить лише елементи HTML (об'єкти типу Element).
      Методи доступу: в основному через індекси та метод namedItem.

    NodeList:
      Може бути живою або статичною, залежно від того, як вона була створена (наприклад, NodeList, отриманий з querySelectorAll, є статичним).
      Містить будь-які типи вузлів, включаючи текстові вузли та елементи.
      Методи доступу: через індекси, а також має метод forEach для ітерації.

    Приклад коду:
      HTMLCollection:
        // Отримання HTMLCollection з усіма параграфами
        const collection = document.getElementsByTagName('p');

        // Ітерація за допомогою циклу for
        for (let i = 0; i < collection.length; i++) {
          console.log(collection[i].textContent); // Виводить текст кожного параграфа
        }

      NodeList:
        // Отримання NodeList з усіма елементами, які мають клас 'example'
        const nodeList = document.querySelectorAll('.example');

        // Ітерація за допомогою методу forEach
        nodeList.forEach(node => {
          console.log(node.textContent); // Виводить текст кожного елемента з класом 'example'
        });
