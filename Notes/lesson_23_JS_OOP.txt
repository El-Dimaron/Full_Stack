Об'єктно-орієнтоване програмування (ООП) — це методологія розробки програмного забезпечення, що використовує об'єкти як основні будівельні блоки. В ООП, програми будуються за допомогою об'єктів, які є інстанціями класів і включають в себе дані (атрибути) та функції для їх обробки (методи).


Основні принципи ООП

  Інкапсуляція
    Інкапсуляція в програмуванні - це процес обмеження доступу до компонентів об'єкта та приховування деталей його внутрішньої реалізації. Ми використовуємо інкапсуляцію для того, щоб розділити інтерфейс об'єкта від його реалізації, дозволяючи змінювати внутрішню логіку без впливу на інші частини програми. Це допомагає нам забезпечити безпеку даних та зробити код більш модульним і легшим для зрозуміння та підтримки.

    Приклад коду, що демонструє інкапсуляцію (без використання класів):
      function createCounter() {
          let count = 0; // 'count' є приватною змінною

          return {
              increment() {
                  count++;
                  return count;
              },
              decrement() {
                  count--;
                  return count;
              },
              getCount() {
                  return count;
              }
          };
      }

  Спадкування
    Спадкування в програмуванні - це механізм, за допомогою якого ми можемо створити новий об'єкт, що базується на існуючому об'єкті. Це дозволяє нам перевикористовувати загальні методи та властивості, уникаючи дублювання коду. Ми використовуємо спадкування для створення ієрархій об'єктів, де дочірні об'єкти наслідують поведінку та характеристики батьківських об'єктів, але також можуть мати свої унікальні властивості та методи.

  Поліморфізм
    Поліморфізм у програмуванні - це концепція, яка дозволяє нам використовувати об'єкти різних типів з однаковим інтерфейсом без необхідності знати конкретний тип об'єкта. Ми використовуємо поліморфізм для того, щоб спростити роботу з різними типами даних, дозволяючи одному й тому ж коду працювати з об'єктами різних класів, які реалізують однакові методи. Це робить наш код більш гнучким і легким для розширення.

    Приклад коду, що демонструє поліморфізм:
      function makeSound(animal) {
      animal.sound();
      }

      const dog = {
          sound() {
              console.log("Гав");
          }
      };

      const cat = {
          sound() {
          console.log("Мяу");
          }
      };

      makeSound(dog); // Виведе "Гав"
      makeSound(cat); // Виведе "Мяу"

    У цьому прикладі ми створюємо два об'єкти: dog та cat. Кожен з цих об'єктів має метод sound, що відповідає за виведення звуку тварини. Функція makeSound приймає об'єкт як параметр і викликає його метод sound. Це демонструє поліморфізм: makeSound може працювати з будь-яким об'єктом, який має метод sound, незалежно від його конкретного типу. Таким чином, ми можемо додавати нові типи тварин, не змінюючи логіку функції makeSound.

  Абстракція
    Абстракція в програмуванні - це процес приховування складних деталей реалізації та показу тільки необхідних аспектів об'єкта користувачу. Ми використовуємо абстракцію для того, щоб спростити розуміння складних систем, розділивши їх на більш прості та зрозумілі компоненти. Це дозволяє нам зосереджуватися на взаємодії на вищому рівні, не заглиблюючись у технічні подробиці.


Клас та Об'єкт в ООП
  В об'єктно-орієнтованому програмуванні клас є шаблоном для створення об'єктів. Він визначає набір атрибутів (властивостей) та методів (функцій), які характеризують будь-який об'єкт цього класу. Класи слугують як інструкції для створення екземплярів (об'єктів), забезпечуючи ініціалізацію їхніх властивостей та реалізацію їхньої поведінки.

  Клас в JavaScript визначається за допомогою ключового слова class та може містити конструктор, методи та статичні методи. Конструктор використовується для ініціалізації нових об'єктів, тоді як методи класу визначають поведінку об'єктів цього класу. Статичні методи викликаються на самому класі, а не на його екземплярах.

  Наприклад:
    class Car {
      constructor(brand, color) {
        this.brand = brand;
        this.color = color;
      }

      drive() {
        console.log(`${this.color} ${this.brand} is driving.`);
      }

      static honk() {
        console.log('Beep beep!');
      }
    }

    const myCar = new Car('Toyota', 'red');
    myCar.drive(); // Виводить: "red Toyota is driving."
    Car.honk(); // Виводить: "Beep beep!"

Прототипи
  Прототипи в JavaScript
  У JavaScript прототипи є основою об'єктно-орієнтованого програмування та механізмом спадкування. Кожен об'єкт має посилання на свій "прототип" — інший об'єкт, з якого він успадковує властивості та методи.

Конструктор
  Функція-конструктор
    Функція-конструктор у JavaScript - це спеціальний тип функції, призначений для створення нових об'єктів. Використовуючи оператор new, ми ініціюємо новий екземпляр об'єкта, при цьому функція-конструктор виконує роль шаблону, який визначає структуру та початковий стан цих об'єктів.

    Назви конструкторів починаються з великої літери, це допомагає відрізнити їх від звичайних функцій.

    Приклад коду, який ілюструє використання функції-конструктора для створення об'єктів, що представляють книги:
      function Book(title, author, year) {
          this.title = title;
          this.author = author;
          this.year = year;

          this.getSummary = function() {
              return `${this.title} написана ${this.author} у ${this.year} році.`;
          };
      }

      // Створюємо два екземпляри книг
      const book1 = new Book('Шлях до вивчення JavaScript', 'Іван Петров', 2021);
      const book2 = new Book('Глибоке занурення в ES6', 'Анна Сидорова', 2023);

      // Виводимо опис першої книги
      console.log(book1.getSummary());
      // Виводимо опис другої книги
      console.log(book2.getSummary());

  Властивість prototype
    Властивість prototype в JavaScript є механізмом, що дозволяє додавати властивості та методи до конструкторів об'єктів. Це означає, що всі екземпляри, створені за допомогою цього конструктора, спільно використовують властивості та методи, визначені в prototype. Це дозволяє ефективно використовувати пам'ять, оскільки кожен екземпляр об'єкта не потребує власної копії цих функцій.

    Розглянемо наступний приклад, де ми оптимізуємо наш попередній код, використовуючи prototype для додавання методу getSummary:

    Помилка
    function Book(title, author, year) {
        this.title = title;
        this.author = author;
        this.year = year;
    }

    // Додаємо getSummary до прототипу Book, щоб усі екземпляри могли використовувати цей метод
    Book.prototype.getSummary = function() {
        return `${this.title} написана ${this.author} у ${this.year} році.`;
    };

  Особливості функції-конструктора
    Конструктори в JavaScript - це звичайні функції з особливими правилами використання:
      - Назва має починатися з великої літери.
      - Вони викликаються з оператором new.
      - Стрілкові функції не можуть слугувати конструкторами.

  instanceof
    Оператор instanceof у JavaScript використовується для перевірки, чи є об'єкт екземпляром певного конструктора. Він дозволяє перевірити, чи входить об'єкт в ланцюг прототипів конструктора, що дає змогу визначити, чи був об'єкт створений за допомогою певної функції-конструктора.

    Приклад:
      console.log(myBook instanceof Book) // true

Основні концепції та Успадкування в класах
  Основні концепції
    Класи в ES6 (ECMAScript 2015) є синтаксичним цукром над існуючим прототипним успадкуванням у JavaScript. Вони надають чіткіший та більш зрозумілий спосіб створення об'єктів та реалізації успадкування. Класи включають конструктор для ініціалізації нових об'єктів, методи класу для додавання функціональності, та можливість використання гетерів та сетерів для керування доступом до властивостей об'єкта.

    Приклад класу ES6:
      class Person {
        // Конструктор для ініціалізації нових об'єктів
        constructor(name, age) {
          this.name = name;
          this.age = age;
        }

        // Метод класу для виведення інформації про особу
        introduceSelf() {
          console.log(`Привіт, мене звати ${this.name} і мені ${this.age} років.`);
        }
      }

      // Створення нового екземпляра класу Person
      const person1 = new Person('Анна', 25);
      person1.introduceSelf(); // Виводить: "Привіт, мене звати Анна і мені 25 років."

  Успадкування в класах
    Успадкування в класах ES6 дозволяє одному класу наслідувати властивості та методи іншого класу. Це реалізується за допомогою ключового слова extends. Успадкування є потужним механізмом для повторного використання коду, дозволяючи дочірньому класу розширювати функціональність батьківського класу. Дочірній клас може використовувати метод super для виклику конструктора батьківського класу та його методів, що дозволяє ініціалізувати батьківський клас або доступатися до його функціональності з дочірнього класу.

    Приклад, який ілюструє успадкування в ES6:
      class Person {
        constructor(name, age) {
          this.name = name;
          this.age = age;
        }

        introduceSelf() {
          console.log(`Привіт, мене звати ${this.name} і мені ${this.age} років.`);
        }
      }

      // Розширюємо клас Person, створюючи новий клас Developer
      class Developer extends Person {
        constructor(name, age, skills) {
          super(name, age); // Викликаємо конструктор батьківського класу
          this.skills = skills; // Додаємо нову властивість специфічну для Developer
        }

        // Додаємо новий метод
        listSkills() {
          console.log(`Мої навички: ${this.skills.join(', ')}`);
        }
      }

      // Створення екземпляра класу Developer
      const developer = new Developer('Олексій', 30, ['JavaScript', 'React', 'Node.js']);
      developer.introduceSelf(); // Виводить: "Привіт, мене звати Олексій і мені 30 років."
      developer.listSkills(); // Виводить: "Мої навички: JavaScript, React, Node.js"

  Cтатичні, приватні та публічні члени
    У JavaScript класи можуть мати статичні, приватні та публічні члени.

    Статичні члени (static) належать самому класу, а не екземплярам класу. Їх можна викликати без створення екземпляра класу. Ми використовуємо статичні члени для функціоналу, який повинен бути доступним на рівні класу, а не на рівні окремих об'єктів.

    Приватні члени класу в JavaScript позначаються символом #. Вони доступні тільки всередині класу, де були оголошені. Ми використовуємо приватні члени для інкапсуляції стану та поведінки, що не повинні бути доступні ззовні класу.

    Публічні члени класу доступні з будь-якого місця, де можна отримати доступ до екземпляра класу. Вони служать для визначення інтерфейсу класу, доступного для зовнішнього використання.

    Приклад коду:
      class MyClass {
          static staticProperty = "Статичне значення"; // Статична властивість
          #privateProperty = "Приватне значення"; // Приватна властивість

          constructor(publicProperty) {
              this.publicProperty = publicProperty; // Публічна властивість
          }

          static staticMethod() {
              return "Статичний метод"; // Статичний метод
          }

          #privateMethod() {
              return "Приватний метод"; // Приватний метод
          }

          publicMethod() {
              return `Публічний метод, ${this.#privateMethod()}`; // Публічний метод
          }
      }

      console.log(MyClass.staticProperty); // Виводить "Статичне значення"
      console.log(MyClass.staticMethod()); // Виводить "Статичний метод"

      const myInstance = new MyClass("Публічне значення");
      console.log(myInstance.publicProperty); // Виводить "Публічне значення"
      console.log(myInstance.publicMethod()); // Виводить "Публічний метод, Приватний метод"
      // Приватні члени не доступні ззовні класу, тому наступні рядки викинуть помилку:
      // console.log(myInstance.#privateProperty); // SyntaxError
      // console.log(myInstance.#privateMethod()); // SyntaxError
