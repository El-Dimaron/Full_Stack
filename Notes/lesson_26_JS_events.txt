Events (Події)
  Події у JavaScript — це сигнали, які повідомляють про те, що щось відбулося у браузері або на веб-сторінці.


Типи подій
  Деякі з основних категорій і приклади подій в кожній з них:

  Події миші:
    click - клік лівою кнопкою миші.
    dblclick - подвійний клік лівою кнопкою миші.
    mouseover - курсор миші наводиться на елемент.
    mouseout - курсор миші виходить за межі елемента.
    mousemove - рух миші над елементом.
    mousedown - натискання кнопки миші на елементі.
    mouseup - відпускання кнопки миші на елементі.

  Події клавіатури:
    keydown - користувач натискає клавішу.
    keyup - користувач відпускає клавішу.
    keypress - вводиться символ (застаріла подія, не рекомендується до використання).

  Події форми:
    submit - надсилання форми.
    change - зміна значення елемента форми.
    focus - елемент форми отримує фокус.
    blur - елемент форми втрачає фокус.

  Події вікна:
    load - завантаження сторінки або зображення.
    resize - зміна розмірів вікна браузера.
    scroll - прокрутка сторінки.

  Події сенсорних екранів:
    touchstart - дотик до сенсорного екрана.
    touchmove - рух пальцем по сенсорному екрану.
    touchend - палець відірваний від сенсорного екрану.

Обробники подій
  Щоб реагувати на різні події у веб-додатках, ми можемо призначати обробники подій за допомогою трьох основних методів:

  1. Використання атрибуту HTML
    Ми можемо додати обробник подій безпосередньо в HTML-код, використовуючи атрибути подій, такі як onclick, onmouseover тощо. Цей метод є найпростішим для реалізації, але він робить код менш гнучким і ускладнює відокремлення HTML від JavaScript.

    <button onclick="alert('Привіт!')">Натисни мене</button>


  2. Використання властивості DOM-об'єкта
    Ми можемо призначити обробники подій, використовуючи властивості об'єктів DOM у JavaScript. Цей метод дозволяє використовувати один обробник для кожної події на елементі, але він є більш гнучким, ніж використання атрибутів HTML.

    const button = document.getElementById('myButton');
    button.onclick = function() {
      alert('Привіт!');
    };


  3. addEventListener (та removeEventListener)
    Це найбільш потужний і гнучкий спосіб призначення обробників подій. Метод addEventListener дозволяє додавати декілька обробників на одну подію, контролювати фазу, в якій обробник має бути викликаний (спливання або захоплення), а removeEventListener дозволяє видаляти обробники, що робить керування подіями ефективним і гнучким.

    const button = document.getElementById('myButton');
    const handleClick = function() {
      alert('Привіт!');
    };
    button.addEventListener('click', handleClick);
    // Щоб видалити обробник, використовуємо:
    button.removeEventListener('click', handleClick);

  Кожен з цих методів має свої переваги та ситуації, в яких він найкраще підходить, дозволяючи розробникам обирати підходящий спосіб в залежності від конкретної задачі.


Об'єкт події Event
  Об'єкт події Event у JavaScript містить інформацію про подію, яка відбулася. Це може бути подія, викликана користувачем, така як клік мишею або натискання клавіші, або системна подія, наприклад завершення завантаження сторінки. Коли обробник події викликається, об'єкт Event передається як аргумент цьому обробнику, надаючи йому деталі про подію, такі як тип події, цільовий елемент, стан клавіш модифікаторів (наприклад, чи була натиснута клавіша Shift) і багато іншого.

  Приклад коду:
    const button = document.getElementById('myButton');

    // Використовуємо addEventListener для призначення обробника події
    button.addEventListener('click', function(event) {
      // Виводимо об'єкт події в консоль
      console.log(event);
    });

Bubbling (Спливання), Capturing (Захоплення), Propagation (Розповсюдження)
  Процес обробки подій у DOM включає три ключові етапи:

  Фаза захоплення (Capturing Phase)
    Подія поширюється з window через DOM вниз до цільового елемента. Це фаза, коли подія "захоплюється" на шляху до цілі.

  Цільова фаза (Target Phase)
    Подія досягає цільового елемента. У цей момент подія обробляється безпосередньо на елементі, який є ціллю події.

  Фаза спливання (Bubbling Phase)
    Після обробки на цільовому елементі, подія "спливає" вгору через DOM до window, проходячи через всі проміжні елементи на своєму шляху.

  Розповсюдження події (Event Propagation) охоплює весь цикл події, від захоплення через цільову обробку до спливання. Метод stopPropagation() може бути використаний на будь-якому етапі цього процесу для запобігання подальшому розповсюдженню події, дозволяючи тим самим контролювати, як подія обробляється і ким.

  Приклад коду:
    // Отримуємо доступ до елементів
    const parentDiv = document.getElementById('parentDiv');
    const childDiv = document.getElementById('childDiv');

    // Обробник для фази захоплення
    parentDiv.addEventListener('click', function() {
      console.log('Клік у батьківському div (захоплення)');
    }, true); // true вказує, що обробник викликається на фазі захоплення

    // Обробник для фази спливання
    childDiv.addEventListener('click', function(event) {
      console.log('Клік у дочірньому div (спливання)');
      event.stopPropagation(); // Запобігаємо подальшому розповсюдженню події
    });

    // Ще один обробник для батьківського div на фазі спливання
    parentDiv.addEventListener('click', function() {
      console.log('Клік у батьківському div (спливання)');
    });

Делегування подій
  Делегування подій — це техніка управління подіями в JavaScript, яка полягає у призначенні одного обробника подій батьківському елементу замість окремого обробника для кожного дочірнього елемента. Це використовує принцип спливання подій у DOM, де події, згенеровані дочірніми елементами, "спливають" до батьківських елементів. Такий підхід дозволяє керувати подіями для динамічно змінюваних елементів (наприклад, для елементів, що додаються до DOM після завантаження сторінки) і зменшує кількість необхідних обробників подій, покращуючи продуктивність та спрощуючи управління кодом.

  Приклад коду:
  Уявімо, що у нас є список елементів, і ми хочемо відстежувати кліки по кожному елементу списку. Замість того, щоб призначати обробник події кожному елементу li індивідуально, ми можемо призначити один обробник подій батьківському елементу ul.
    <ul id="myList">
      <li>Пункт 1</li>
      <li>Пункт 2</li>
      <li>Пункт 3</li>
    </ul>

    // Отримуємо доступ до батьківського елементу
    const list = document.getElementById('myList');

    // Призначаємо обробник подій батьківському елементу
    list.addEventListener('click', function(event) {
      // Перевіряємо, чи клік було здійснено по елементу li
      if (event.target.tagName === 'LI') {
        alert(`Клікнуто на ${event.target.textContent}`);
      }
    });
