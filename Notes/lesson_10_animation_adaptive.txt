CSS анімація

Базова анімація - це transition
transition = анімований міст між двома станами стилю

transition — це властивість, яка дозволяє плавно анімувати зміну стилів між двома станами елемента.
  Найчастіше використовується для hover-ефектів, фокусів, активних станів кнопок, карток, меню тощо.

Властивість transition - це скорочення від наступних властивостей:

  transition-property
    sets the CSS properties to which a transition effect should be applied.

  transition-duration
    sets the length of time a transition animation should take to complete. By default, the value is 0s, meaning that no animation will occur.

  transition-timing-function
    sets how intermediate values are calculated for CSS properties being affected by a transition effect.
    Default is linear, but any other over-time-changes settings can be set up.

  transition-delay
    duration to wait before starting a property's transition effect when its value changes

  transition-behavior
    Not widely used


Базовий синтаксис transition:
  transition: property duration timing-function delay;

    Example:
      transition: background 0.3s ease;
        При наведенні колір плавно зміниться за 0.3 секунди, замість різкого стрибка.

Важливі нюанси
  ✔ transition працює тільки коли значення змінюється
  ✔ Не всі властивості анімуються (наприклад display — ні)
  ✔ Для продуктивності краще анімувати:
    - transform
    - opacity
      (вони використовують GPU і не лагатимуть)



Анімації у CSS зазвичай виконуються за допомогою двох основних компонентів:

  @keyframes
    Правило, яке визначає анімацію, задаючи стилі для різних моментів анімації (наприклад, початок, середина та кінець).
  animation
    Властивість, яка застосовує анімацію до елемента, задаючи тривалість, затримку, кількість повторень та інші параметри.


@keyframes
  Дозволяє створити анімацію, визначаючи стилі для елемента в різні моменти часу.
  Кожен момент анімації, або "кадр" (keyframe), задається у відсотках від загальної тривалості анімації, де 0% - це початок, а 100% - кінець анімації.


Приклад:
  @keyframes example {
    0%   { background-color: red; left: 0; top: 0; }
    25%  { background-color: yellow; left: 200px; top: 0; }
    50%  { background-color: blue; left: 200px; top: 200px; }
    75%  { background-color: green; left: 0; top: 200px; }
    100% { background-color: red; left: 0; top: 0; }
  }

  div {
    width: 100px;
    height: 100px;
    position: absolute;
    animation: example 5s infinite;
  }


animation - це CSS властивість, яка дозволяє застосовувати анімації до HTML елементів.
  Вона дає можливість контролювати тривалість анімації, її затримку, кількість повторень, напрямок та спосіб анімації.

Як використовується?
  Ця властивість зазвичай використовується разом з @keyframes, де визначається сама анімація. Властивість animation дозволяє нам визначити, як і коли анімація має бути відтворена. Ми можемо задати кілька параметрів, таких як:
    animation-name: ім'я анімації, визначеної в @keyframes.
    animation-duration: тривалість анімації.
    animation-timing-function: темп анімації (наприклад, лінійний, із затримкою).
    animation-delay: затримка перед початком анімації.
    animation-iteration-count: кількість повторень анімації.
    animation-direction: напрямок анімації.
    animation-fill-mode: вказує, як стилі, задані анімацією, застосовуються до елемента перед та після анімації.
    animation-play-state: стан анімації, наприклад, програється чи на паузі.




Viewport
Viewport - це видима область веб-сторінки, яку можна побачити в браузері.
  Розмір viewport залежить від розміру пристрою, на якому відкривається веб-сторінка, що особливо важливо при розробці відгукових (responsive) веб-сайтів. Мета-тег viewport використовується у HTML для контролю масштабування та розмірів viewport на мобільних пристроях.

У HTML файлі можна визначити налаштування viewport таким чином:
  <head>
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
  </head>

У цьому прикладі, мета-тег viewport встановлює ширину видимої області (viewport) такою ж, як ширина пристрою, на якому вона відображається, і встановлює початковий масштаб сторінки (initial-scale) рівним 1.0. Це забезпечує, що веб-сайт буде правильно відображатися на різних пристроях, адаптуючись до їх розмірів.


Media Queries (Медіа-запити)
  Це функція CSS, що дозволяє нам застосовувати стилі до веб-сторінки залежно від характеристик пристрою, на якому вона переглядається, таких як ширина екрану, висота, роздільна здатність і т.д.


CSS правило @media
  У практиці розробки веб-сайтів, часто використовуються медіа-запити в CSS, включаючи параметри як @media screen, min-width, max-width, min-height, та max-height.

Ці параметри дозволяють нам вказувати, при яких умовах певні стилі CSS мають бути застосовані.
  @media screen вказує, що стилі будуть застосовані до екранних пристроїв (наприклад, моніторів, телефонів, планшетів).
  Функції min-width та max-width визначають мінімальну та максимальну ширину вікна перегляду, при якій стилі будуть застосовані.
  Аналогічно, функції min-height та max-height встановлюють мінімальну та максимальну висоту.


  Приклад:
    body {
      background-color: lightblue;
    }

    @media screen and (min-width: 600px) {
      body {
        background-color: lightgreen;
      }
    }

Screen
  У багатьох випадках, коли не потрібно спеціально налаштовувати стилі для різних типів медіа (як друк або синтезатори мовлення), можна опустити вказівку screen у медіа-запитах. Це спрощує код і зосереджує увагу на стилізації для екранних пристроїв, які є основним середовищем перегляду веб-сторінок.



Breakpoints (Точки перелому)
  Це певні значення ширини (іноді висоти) екрану, при яких ми змінюємо стилі веб-сторінки для кращого відображення на різних пристроях. Ці значення визначаються в CSS за допомогою медіа-запитів. Вони важливі для створення адаптивного дизайну, який забезпечує оптимальний вигляд та функціональність сайту на будь-якому пристрої.

  Точки перелому дозволяють адаптувати макет, розмір тексту, зображення та інші елементи дизайну до різних розмірів екранів. Використання точок перелому сприяє створенню більш гнучкого та користувацького інтерфейсу, підвищуючи загальну доступність та зручність веб-сайту.
